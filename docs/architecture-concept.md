# Техническое описание архитектуры NeuroAssistant

NeuroAssistant  - это программный комплекс, в основу которой лежит широкий интерфейс настроек для создания собственного AI агента на базе Python, а также готовые инструменты для повышения продуктивности на основе инструментов органайзера, использующие предсказания текста для записи заметок, событий, планирования, и так далее. В таком варианте исполнения нейросеть выполняет роль предсказателя на двух этапах:

- формирование сути задачи, то есть каркаса вопросов для поиска ответов
- проверки полноты покрытия изначального вопроса или под-вопроса через анализ промежуточного результата
- формирование ключевых разделов каркаса будущего решение
- итеративный синтез ответа или промежуточного результата, используя подготовленный каркас

Суть - создать настраиваемую базу AI агента для итеративного поиска, планирования, а также наполнения конечного результата, используя локальный SLM для генерации простых текстовых команд и любой LLM исключительно для связности смыслов в тексте.

Цель - декомпозировать шаги исследования и конечной генерации на более низком уровне с сохранением промежуточных результатов работы в кэше и постоянной памяти, а также автоматизировать процесс, тем самым выяснив минимальный необходимый объем LLM модели для такой работы.

## Общее описание

Представим, что мы наблюдаем над картиной в музее. Личность не только опишет ее словестно, но и расскажет, что думает, а также интересные факты. Каких-то знаний может не быть (как долго картину рисовали, какие краски были, процесс работы) - это глубокие знания, которые не нужны для описания того, что нужно сказать о картине. Вместо этого человек скажет, что чего-то не знает, а знания в том или ином случае приходит из разных областей:

- культуры и науки, формируя ветку фундаментальных знаний
- практического опыта и применения

Попытаемся представить шаги мышления и решения задачи, когда задается вопрос:

- личность и задача
- описание картинки с учетом личности
- знания о текущей картинке
- какая информация о картинке поверхностная, а какая глубокая
- (сигнал к действию по отношению к картинке)
- описание, что мы делаем с картинкой?
- поиск недостающей информации о ней
- поиск дополнительной информации, помогающей решить текущую задачу
- проверка знаний, чего не хватает, риски, уверенность, лимит, дальнейшие действия
- повторные циклы поиска, пока знаний не будет хватать
- систематизация накопленных теоретических и практических знаний
- оценка текущей полноты информации и рисков
- продумывание плана текущей работы
- рекурсивная декомпозиция работы на атомарно возможные шаги, выделяя углубленные пробелы для дальнейшего исследования
- следование плану рекурсивно шаг за шагом

## Ключевые компоненты проекта

### NeuroBIOS

Входная точка проекта. Оркестратор, координирующий работу системы анализа и поиска информации, используя комбинацию SLM для вероятностных вычислений качественных метрик и LLM для синтеза текста исключительно из исходного контекста.

### NeuroBus

Протокол передачи данных между модулями системы на уровне текстовых команд: SLM (логика) <-> система <-> LLM (мышление)

### NeuroStorage (in-memory и постоянное хранилище)

Хранилище накопленной информации в виде теоретических знаний и навыков практического применения.

- NeuroTree - базовая древовидная структура данных в системе, представляющая собой одну область знаний
- NeuroMemories - быстрое хранилище ключевой информации
- NeuroStrategyMap - постоянная карта решения задачи с описанием метрик
- NeuroDB - постоянное хранилище знаний и фактов

### NeuroKernel

Центральное ядро системы вероятностных предсказаний в виде качественных метрик.

В системе может участвовать сколь угодно ядер при условии регистрации инструкций в системе

Базовая логика:

- NeuroCore - базовое настраиваемое ядро вероятностных предсказаний на базе FunctionGemma

Подключаемая логика (middleware):

- NeuroCompilerModule - преобразование текста в команды
- NeuroRuntimeModule - преобразование команд в текст
- NeuroStorageModule - работа с хранилищем данных

#### NeuroLogicKernel

Основано на NeuroKernel на базе FunctionGemma иди иной SLM для синтеза команд и аргументов

Базовая логика:

- NeuroSearchCore - поиск информации в текущем контексте

Подключаемая логика (middleware):

- NeuroAssessmentModule - предсказание качественных метрик
- NeuroSearchStorageModule - текущий контекст поиска знаний
- NeuroSearchTaskManagerModule - планировщик наполнения информацией

#### NeuroSpeechKernel

Основано на NeuroKernel на базе внешней подключаемой LLM для синтеза текста

Базовая логика:

- NeuroPlanCore - наполнение и уточнение шаблона результата работы
- NeuroPlanStorage - контекст рассуждений, принятых решений, а также дальнейших шагов
- NeuroPlanTaskManager - планировщик задач

Подключаемая логика (middleware):

- NeuroResultConstructionModule - наполнение и уточнение шаблона исследования
- NeuroResultRenderModule - поэтапное наполнение шаблона содержимым

#### NeuroTaskManagerKernel
Протокол управления командами на основе приоритетов

- Управление текущим контекстом
- рамки работы (лимиты)
- приоретизация задач
- обработка исключений 
- Монитор системы, команд в очереди, потребляемых ресурсов системы
- Текущая конфигурация и проверка статуса компонентов
- Логи работы, ошибок, исключений (уточнений от пользователя)

## Предсказываемые команды

Примерный список команд системы "Процессор" - связки SLM модели с обособленным оркестратором на Python.
Для удобства разделим на группы.

Команды сканирования контекста

- OBSERVE (сканирование дерева контекста задачи: где мы сейчас и что делаем?)
- SCAN (сканирование дерева решений для получения контекста)
- ANALYZE (что уже знаем из дерева и что нам надо делать дальше? надо ли искать еще глубже в текущей теме или уже можно синтезировать ответ? предсказание аргументов и адреса памяти, оценка качества полученной информации: отвечает ли на вопрос? есть ли пробелы в полученной информации? что еще нужно поискать?)
- CHECK (что мы уже знаем для задачи и что нам надо делать дальше?)
- PREDICT (предсказание качественных метрик словами HIGH, MEDIUM, LOW)
- VERDICT (GO/DROP)

Команды работы с деревом решений

- EXPAND (запуск процесса углубления поиска всех ветвей поиска знаний)
- CONCLUDE (синтез конечного результата из дерева решений)
- EXCEPTION (найдены пробелы, нужен OBSERVE)
- ESCALATE (требуется вмешательство пользователя)
- GROUND (достигнут необходимый уровень поиска)
- DEAD (мертвая ветвь поиска - не учитывать при сканировании)

Работа с циклом через планировщик задач через команды процессора

- BEGIN (начало цепочки команд)
- DONE (конец цепочки команд)

## Конечные продукты на базе NeuroAssistant:

- NeuroNotes - умная записная книжка, использующая изначальный контекст для предсказания структуры заметок и наполнения содержимым из предоставленных файлов (синтез из контекста)
- NeuroCalendar - предсказатель планов эффективной стратегий на будущее с учетом заметок (система приоритетов)
- NeuroSearch - нейросетевой поисковой движок в интернете и на основе файлов пользователя
